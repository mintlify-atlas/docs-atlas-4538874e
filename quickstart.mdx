# Quickstart Guide

This guide will help you get HozonDB up and running in just a few minutes.

## Prerequisites

Make sure you have Rust and Cargo installed. See the [Installation](/installation) page for setup instructions.

## Running HozonDB

<Steps>
  <Step title="Clone and build">
    Navigate to the HozonDB source directory and run:
    
    ```bash
    cargo run
    ```
    
    You'll see the HozonDB REPL start:
    
    ```
    HozonDB v0.1.0
    Enter '.help' for usage hints.
    
    hozondb>
    ```
  </Step>
  
  <Step title="Open a database file">
    Before executing SQL commands, you need to open or create a database file:
    
    ```sql
    hozondb> .open test.hdb
    Opened database file: test.hdb
    ```
    
    <Note>
    If the file doesn't exist, HozonDB will create it automatically. Database files use the `.hdb` extension.
    </Note>
  </Step>
  
  <Step title="Create your first table">
    Let's create a simple users table:
    
    ```sql
    hozondb> CREATE TABLE users (id INTEGER, name TEXT);
    Table 'users' created.
    ```
    
    HozonDB supports these data types:
    - `INTEGER` - 32-bit signed integers
    - `TEXT` - Variable-length strings
    - `BOOLEAN` - True/false values
    - `NULL` - Null values (can be used in any column)
  </Step>
  
  <Step title="Insert data">
    Add some rows to your table:
    
    ```sql
    hozondb> INSERT INTO users VALUES (1, 'Alice');
    1 row inserted.
    
    hozondb> INSERT INTO users VALUES (2, 'Bob');
    1 row inserted.
    
    hozondb> INSERT INTO users VALUES (3, 'Charlie');
    1 row inserted.
    ```
  </Step>
  
  <Step title="Query your data">
    Retrieve all rows:
    
    ```sql
    hozondb> SELECT * FROM users;
    | id | name |
    | 1 | "Alice" |
    | 2 | "Bob" |
    | 3 | "Charlie" |
    ```
    
    Or select specific columns:
    
    ```sql
    hozondb> SELECT name FROM users;
    | name |
    | "Alice" |
    | "Bob" |
    | "Charlie" |
    ```
  </Step>
</Steps>

## Using WHERE Clauses

HozonDB supports filtering with WHERE clauses:

```sql
-- Equality comparison
hozondb> SELECT * FROM users WHERE id = 2;
| id | name |
| 2 | "Bob" |

-- Greater than
hozondb> SELECT * FROM users WHERE id > 1;
| id | name |
| 2 | "Bob" |
| 3 | "Charlie" |

-- Text comparison
hozondb> SELECT * FROM users WHERE name = 'Alice';
| id | name |
| 1 | "Alice" |
```

### Supported Operators

- **Comparison**: `=`, `>`, `<`, `>=`, `<=`
- **Logical**: `AND`, `OR`

Example with logical operators:

```sql
hozondb> SELECT * FROM users WHERE id > 1 AND id < 3;
| id | name |
| 2 | "Bob" |
```

## Updating and Deleting Data

### UPDATE Statement

Modify existing rows:

```sql
hozondb> UPDATE users SET name = 'Alice Smith' WHERE id = 1;
1 row updated.
```

Update multiple rows:

```sql
hozondb> UPDATE users SET name = 'Anonymous' WHERE id > 1;
2 rows updated.
```

### DELETE Statement

Remove specific rows:

```sql
hozondb> DELETE FROM users WHERE id = 3;
1 row deleted.
```

Delete all rows matching a condition:

```sql
hozondb> DELETE FROM users WHERE id > 1;
2 rows deleted.
```

## Meta Commands

HozonDB provides several meta-commands (commands starting with `.`):

<AccordionGroup>
  <Accordion title=".help - Show available commands">
    ```
    hozondb> .help
    Available commands:
      .help              - Show this help message
      .open <file>       - Open or create a database file
      .benchmark [rows]  - Run performance benchmarks (default: 10000 rows)
      .exit              - Exit the program
    ```
  </Accordion>
  
  <Accordion title=".open <file> - Open a database">
    Opens an existing database file or creates a new one:
    ```
    hozondb> .open mydata.hdb
    Opened database file: mydata.hdb
    ```
    
    <Warning>
    You must open a database before executing SQL commands. Only one database can be open at a time.
    </Warning>
  </Accordion>
  
  <Accordion title=".benchmark [rows] - Run benchmarks">
    Runs performance tests on your database:
    ```
    hozondb> .benchmark 1000
    
    ╔══════════════════════════════════════════════════════════════╗
    ║              HozonDB Benchmark Starting...                   ║
    ╚══════════════════════════════════════════════════════════════╝
    
    Configuration:
      Rows: 1000
    
    Setting up test data...
    Running benchmarks...
    Cleaning up...
    ```
    
    The benchmark suite tests:
    - **Read operations**: Full table scan, first/middle/last row selects
    - **Write operations**: Single and bulk inserts, updates, and deletes
    
    Results include:
    - Duration (ms)
    - Pages read/written
    - Rows scanned/modified
  </Accordion>
  
  <Accordion title=".exit - Exit the REPL">
    Exits HozonDB and closes the database:
    ```
    hozondb> .exit
    Exiting HozonDB. Goodbye!
    ```
    
    You can also use `.quit` as an alias.
  </Accordion>
</AccordionGroup>

## Complete Example Session

Here's a complete example showing all major features:

```sql
# Start HozonDB
$ cargo run
HozonDB v0.1.0
Enter '.help' for usage hints.

hozondb> .open example.hdb
Opened database file: example.hdb

# Create table with multiple data types
hozondb> CREATE TABLE products (id INTEGER, name TEXT, price INTEGER, in_stock BOOLEAN);
Table 'products' created.

# Insert some products
hozondb> INSERT INTO products VALUES (1, 'Laptop', 999, true);
1 row inserted.

hozondb> INSERT INTO products VALUES (2, 'Mouse', 25, true);
1 row inserted.

hozondb> INSERT INTO products VALUES (3, 'Keyboard', 75, false);
1 row inserted.

# Query all products
hozondb> SELECT * FROM products;
| id | name | price | in_stock |
| 1 | "Laptop" | 999 | true |
| 2 | "Mouse" | 25 | true |
| 3 | "Keyboard" | 75 | false |

# Find products in stock
hozondb> SELECT name, price FROM products WHERE in_stock = true;
| name | price |
| "Laptop" | 999 |
| "Mouse" | 25 |

# Update prices
hozondb> UPDATE products SET price = 20 WHERE name = 'Mouse';
1 row updated.

# Delete out-of-stock items
hozondb> DELETE FROM products WHERE in_stock = false;
1 row deleted.

# Drop the table when done
hozondb> DROP TABLE products;
products table successfully dropped

hozondb> .exit
Exiting HozonDB. Goodbye!
```

## Tips and Best Practices

<Tip>
**Data Persistence**: All data is automatically saved to disk after each operation. You can safely exit and reopen your database files.
</Tip>

<Tip>
**Keywords are case-insensitive**: You can use `SELECT`, `select`, or `Select` - they all work the same way.
</Tip>

<Warning>
**File Locking**: HozonDB uses file-based locking to prevent concurrent access. Only one process can open a database file at a time.
</Warning>
