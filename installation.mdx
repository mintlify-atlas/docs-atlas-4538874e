# Installation

HozonDB is built in Rust and requires Cargo (Rust's package manager) to build and run.

## Prerequisites

### Install Rust

If you don't have Rust installed, the easiest way is using `rustup`:

<CodeGroup>
```bash macOS/Linux
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

```powershell Windows
# Download and run rustup-init.exe from https://rustup.rs
```
</CodeGroup>

After installation, verify that Rust is installed:

```bash
rustc --version
cargo --version
```

You should see output like:

```
rustc 1.75.0 (or later)
cargo 1.75.0 (or later)
```

<Note>
HozonDB uses Rust edition 2024. Make sure you have a recent version of Rust (1.75.0 or later recommended).
</Note>

## Installing HozonDB

<Steps>
  <Step title="Clone the repository">
    First, clone the HozonDB source code:
    
    ```bash
    git clone https://github.com/yourusername/hozondb.git
    cd hozondb
    ```
    
    <Info>
    Replace `yourusername` with the actual repository location.
    </Info>
  </Step>
  
  <Step title="Build the project">
    Build HozonDB using Cargo:
    
    ```bash
    cargo build --release
    ```
    
    This will compile HozonDB in release mode with optimizations. The first build may take a few minutes.
    
    <Tip>
    The `--release` flag enables optimizations for better performance. For development, you can omit it to build faster in debug mode.
    </Tip>
  </Step>
  
  <Step title="Run HozonDB">
    You can run HozonDB directly using Cargo:
    
    ```bash
    cargo run --release
    ```
    
    Or use the compiled binary:
    
    ```bash
    ./target/release/hozondb
    ```
    
    You should see the HozonDB REPL start:
    
    ```
    HozonDB v0.1.0
    Enter '.help' for usage hints.
    
    hozondb>
    ```
  </Step>
</Steps>

## Project Structure

HozonDB has a clean, modular architecture with zero external dependencies:

```
hozondb/
├── Cargo.toml              # Project manifest (edition 2024)
├── README.md               # Project overview
└── src/
    ├── main.rs             # Entry point
    ├── lib.rs              # Library root
    ├── repl.rs             # Interactive REPL
    ├── storage/            # Storage engine
    │   ├── mod.rs
    │   └── page.rs         # 4KB page manager
    ├── catalog/            # Schema management
    │   ├── mod.rs
    │   ├── table.rs        # Table catalog
    │   ├── schema.rs       # Table schemas
    │   └── row.rs          # Row serialization
    ├── sql/                # SQL parsing & execution
    │   ├── mod.rs
    │   ├── tokenizer.rs    # SQL tokenizer
    │   ├── parser.rs       # SQL parser
    │   ├── executor.rs     # Query executor
    │   └── evaluator.rs    # Expression evaluator
    └── benchmark/          # Performance testing
        ├── mod.rs
        ├── runner.rs       # Benchmark runner
        └── metrics.rs      # Performance metrics
```

```toml Cargo.toml

name = "hozondb"
version = "0.1.0"
edition = "2024"

[dependencies]
# No external dependencies - built from scratch!

# Test dependencies would go here
```

HozonDB is built entirely from scratch with **zero external dependencies**. Everything from the storage engine to the SQL parser is implemented in pure Rust.
</Note>

## Building from Source

### Development Build

For development work with faster compile times:

```bash
cargo build
cargo run
```

### Release Build

For production use with full optimizations:

```bash
cargo build --release
./target/release/hozondb
```

### Running Tests

HozonDB includes comprehensive unit tests:

```bash
# Run all tests
cargo test

# Run tests with output
cargo test -- --nocapture

# Run specific test module
cargo test repl
cargo test storage::page
```

The test suite covers:
- Page manager operations (allocation, free list management)
- SQL parsing and execution
- REPL commands
- Data persistence
- WHERE clause evaluation
- Multi-page table operations

## Database Files

HozonDB creates two types of files:

### Database File (`.hdb`)

The main database file containing all your data:

- Organized in 4KB pages
- Page 0: Database header (magic number, page count, free list)
- Page 1: Table catalog
- Page 2+: Table data pages

```
test.hdb         # Your database file
```

### Lock File (`.hdb.lock`)

Prevents concurrent access to the database:

- Created when database is opened
- Automatically deleted when closed
- Ensures single-writer access

```
test.hdb.lock    # Lock file (auto-managed)
```

<Warning>
**Concurrent Access**: HozonDB uses file-based locking to prevent data corruption. If you see a "Database is already in use" error, make sure no other process has the database open.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database is already in use">
    This error occurs when a lock file exists from a previous session:
    
    ```
    Error: Database is already in use. Lock file exists: test.hdb.lock
    ```
    
    **Solution**: Make sure no other HozonDB process is using the file. If the process crashed, manually delete the `.hdb.lock` file:
    
    ```bash
    rm test.hdb.lock
    ```
  </Accordion>
  
  <Accordion title="Rust edition 2024 errors">
    If you see errors about edition 2024:
    
    ```
    error: edition 2024 is unstable
    ```
    
    **Solution**: Update your Rust toolchain to the latest version:
    
    ```bash
    rustup update stable
    ```
  </Accordion>
  
  <Accordion title="Build fails with linking errors">
    If you encounter linking errors during build:
    
    **Solution**: Make sure you have the required build tools installed:
    
    - **Linux**: `build-essential`, `pkg-config`
    - **macOS**: Xcode Command Line Tools
    - **Windows**: Visual Studio Build Tools
  </Accordion>
</AccordionGroup>

## Development Setup

For contributors and developers:

```bash
# Clone repository
git clone https://github.com/yourusername/hozondb.git
cd hozondb

# Install Rust (if needed)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Build and test
cargo build
cargo test

# Run in development mode
cargo run

# Check code formatting
cargo fmt --check

# Run clippy (linter)
cargo clippy
```

**IDE Setup**: HozonDB works great with:
